# DEV LOG - 2026-05-03

## Summary

Today we improved Composite node bounding behavior when a Shape mask is connected.
Previously, the selection box and transform handles covered the entire canvas (full inset:0 wrap).
Now they snap to the mask's actual footprint, making it easy to transform or manipulate the masked composite as a unit.

## Main Work Completed

### 1. Composite mask bounding box fix

**Problem:**
When a Shape node was wired into a Composite's `mask` input, the selection box
and `measureLayerBounds` result covered the full canvas, because the output
element used `position: absolute; inset: 0` with no `data-bound-id` marker.
This made transforms and downstream selections imprecise.

**Fix:**
In `scripts/nodes/composite.js`, after the mask is applied, if `maskMeta.shape`
resolves to a simple shape (circle, rectangle, star, etc. — not a pen path),
a hidden `<div>` is appended to the output wrap:

```js
bounds.dataset.boundId = node.id;
bounds.style.position = "absolute";
bounds.style.left = (bx - bw / 2) + "px";
bounds.style.top  = (by - bh / 2) + "px";
bounds.style.width  = bw + "px";
bounds.style.height = bh + "px";
bounds.style.visibility = "hidden";
bounds.style.pointerEvents = "none";
```

This marker is used by:
- `updatePreviewSelectionBox` — draws the selection outline around the mask region.
- `measureLayerBounds` — export / downstream transform sizing.

**Exclusion:**
Pen path (shapeType `"path"`) shapes are excluded for now because computing a
tight bounding box over arbitrary SVG path data needs extra work.
