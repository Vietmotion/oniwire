# DEV LOG - 2026-04-22

## Summary

Today we added and refined a full Stylize pipeline, improved gradient editing UX, and fixed a transform-chain masking scope bug.

Result:
- New `Stylize` node is integrated in the graph and menu.
- Stylize modes are now: `Lookup filter`, `Brutal`, `Glitch`.
- `Brutal` uses real gradient-map behavior (tone remap), not overlay tinting.
- Ramp node inspector now supports a draggable gradient slider workflow.
- Mask/bounds scoping is stable through `Composite -> Transform -> Stylize`.

## Main Work Completed

1. Stylize node creation and wiring
- Added new node file: `scripts/nodes/stylize.js`.
- Registered in node defs (`scripts/nodes.js`) and menu (`scripts/node-menu.js`).
- Added script load and tips/category updates in `oniwire_v0_2_3.html`.

2. Stylize mode architecture
- Consolidated style looks under `Lookup filter` mode.
- Kept `Glitch` as a standalone mode for future deeper controls.
- Renamed `Brutalism` to `Brutal` and made it a separate mode.
- Added migration handling for old preset values.

3. Brutal mode rendering
- Implemented proper gradient-map remap using SVG `feComponentTransfer` table lookup.
- Added draggable 3-stop gradient control for shadow/mid/highlight mapping.
- Added grain control and preserved vignette behavior.

4. Ramp node inspector UX
- Replaced per-stop position sliders with one gradient slider track + draggable handles.
- Preserved add/remove stop operations and color editing.
- Improved add-stop insertion logic by placing new stop in the largest gap.

5. Transform-chain mask scope fix
- `Transform` now always emits transformed bounds markers (not only when selected).
- `Stylize` now resolves mask host across descendants and inherits mask config robustly.
- Fixed spill where Stylize affected space outside masked output in transform chains.
