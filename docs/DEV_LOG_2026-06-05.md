## Group System — Collapsed Wire Fix

### Problems
- Wires to/from collapsed group members flew far away instead of anchoring to the group edge.
- Root cause: group rect was in world-space but wire endpoints are in editor-space (pan+zoom applied).

### Fixes
- `getGroupAnchorRect()` added to return the visible collapsed height (header bar) instead of full stored height.
- `getAnchorPointOnGroup()` now converts group rect from world-space to editor-space before edge intersection math.
- Outgoing wires from a collapsed group are forced to right edge center; incoming to left edge center.

## Pen Node — Full Inspector Overhaul

### Removed
- Shortcut tutorial block removed from the `shapeManager` inspector (already in Tips panel).
- Redundant "Stroke Style" section header and duplicate global Line Cap / Taper dropdowns removed.
- Separate `Taper = None/Start/End/Both` dropdown removed entirely.

### New graph inputs
- `fill` input port: plug a Color or Gradient node to override fill color for all paths.
- `stroke` input port: same for stroke color. Linear/radial gradient metadata is converted to SVG paint servers automatically.

### True width taper (replaced opacity fade)
- Taper is now geometric: the stroke is rendered as a variable-width ribbon built from sampled Bezier points.
- Width narrows toward the tapered end(s) instead of fading opacity.
- `taperStartStrength` (0–100) and `taperEndStrength` (0–100) control each end independently.
- If both are 0 → no taper. Any nonzero value activates that end automatically.
- Taper span scales with strength: at 100 the effect covers ~55% of the path from that end.

### Smooth Strength
- New per-path `smoothStrength` (0–100) parameter.
- Smooth button uses this when computing Catmull-Rom handle positions.
- Low = subtle curve adjustment; high = very smooth flowing curves.

### Inspector layout
- Fill and Stroke sections visually separated with section labels.
- Number input + slider pairs for Fill Opacity, Stroke Width, Stroke Opacity, Smooth Strength.

## Figma-Style Scrub Button Pattern (`smScrubBox`)

This is a reusable pattern introduced for the Taper controls. It will be used again.

### HTML structure
```html
<div class="smScrubRow">
  <button class="smScrubBox" data-sm-scrub="propName" data-sm-min="0" data-sm-max="100" data-sm-step="1">
    <span class="smScrubTag">S</span>
    <span class="smScrubValue">70</span>
  </button>
</div>
```

### CSS classes (styles.css)
- `.smScrubRow` — flex row container for the buttons
- `.smScrubBox` — the draggable pill button with `cursor: ew-resize; touch-action: none;`
- `.smScrubTag` — small mono label (S / E)
- `.smScrubValue` — the numeric value shown in the box
- `.smScrubBox.isMuted` — dims the button when inactive

### JS handler pattern (shapeManager event block in renderInspector)
Key lessons learned:
- `ap = getActivePath()` goes stale after `evaluateAndRender()` because `normalizePaths()` inside `run()` replaces `node.params.paths` with fresh objects.
- Fix: use `getLivePath()` that re-reads from `node.params.paths[activePath]` on every render frame.
- Use `pointermove` / `pointerup` on the button itself with `setPointerCapture` — more reliable than `window.mousemove`.
- Update the visible number (`valueEl.textContent`) directly on every `pointermove`, independent of the render frame.
- Use `requestAnimationFrame` to throttle the heavy `evaluateAndRender()` call.
- Sensitivity: accumulate fractional movement, no `carry / 2` divisor.
- `Shift` while dragging = fine mode (0.2× speed).

### Data flow
1. `pointermove` → accumulate movement → update `currentValue` + `valueEl.textContent` immediately
2. `requestAnimationFrame` → call `getLivePath()` fresh → write `currentValue` → `evaluateAndRender()`
3. `pointerup` → cancel pending raf → final write + render + `pushHistory()`
