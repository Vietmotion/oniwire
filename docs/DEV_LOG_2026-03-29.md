# DEV LOG - 2026-03-29

## Summary

Today was mainly about closing the gap between editor preview behavior and MP4 export behavior for a masked Composite graph using Image + Text + Pen + Transform + Motion.

End state:

- Pen output can now drive Composite/Mask masking in the live editor.
- MP4 export now preserves the masked image instead of dropping it.
- Export brightness now matches preview more closely by treating Pen mask opacity as binary visibility, like the live viewport.
- Motion export for the masked composite now behaves correctly by baking the mask on the Composite layer itself instead of applying a late full-frame mask.

## What changed

### 1. Pen can act as a real mask source

- Added Pen path serialization in `scripts/nodes/pen.js`.
- Pen now writes mask metadata onto its rendered wrapper element:
  - `data-mask-shape="path"`
  - `data-mask-path-data`
  - transform context (`maskX`, `maskY`, `maskScale`, `maskCx`, `maskCy`)
- Extended `resolveMaskMetaFromEl()` in both app entry files so path masks are detected and reconstructed.
- Extended `buildShapeMaskUrl()` so path masks can be turned into SVG mask markup for live CSS masking.

### 2. Compact/export graph player got Pen rendering

- Added full Pen rasterization support to `scripts/export.js`.
- New helpers include:
  - `clampPenValue`
  - `normalizePenPoints`
  - `defaultPenPath`
  - `normalizePenPaths`
  - `migrateLegacyPenParams`
  - `tracePenPath`
- Added a `Pen` branch in `CompactGraphPlayer.renderNode()` so canvas-based export/runtime paths understand Pen nodes.

### 3. MP4 export image loading was fixed

- Discovered MP4 export does not use `CompactGraphPlayer`; it clones DOM from `renderRoot` and snapshots it.
- Added `waitForRenderableImages()` in both `oniwire_v0_2_2.html` and `index.html`.
- Snapshot paths now wait for cloned `<img>` elements to decode before rendering.
- Added `crossOrigin = "anonymous"` for remote HTTP(S) images in:
  - `scripts/nodes/image.js`
  - `scripts/export.js`

### 4. Export masking was rebuilt around metadata, not fragile DOM scraping

- `scripts/nodes/composite.js` now stamps resolved mask metadata onto the Composite output wrapper itself.
- Export masking no longer depends only on reading `mask-image` back out of computed style.
- Added mask raster helpers in both app entry files:
  - `canvasHasOpaquePixels()`
  - `drawShapeMaskToContext()`
  - `buildLuminanceMaskCanvas()`
- Pen masks are rasterized directly to canvas from stored path metadata with `Path2D`.

### 5. Export mask behavior now matches viewport behavior better

- The live viewport treats visible Pen paths as effectively full-strength mask visibility.
- The export path originally respected `fillOpacity` / `strokeOpacity` as alpha, which darkened the exported image.
- Changed export mask rasterization to use full alpha for visible Pen fills/strokes so preview and MP4 match.

### 6. Motion/pivot mismatch was fixed by changing where mask baking happens

- The graph was: `Composite -> Transform -> Motion -> Output`.
- A late full-frame post-mask caused wrong pivot behavior because the masked result no longer moved as one transformed unit.
- Export now bakes the mask at the Composite element level inside the cloned DOM before final frame capture.
- To avoid double-transform issues, the masked Composite is rendered from an isolated offscreen clone with ancestor transforms removed.
- Result: the Composite behaves as a single unit under downstream Transform/Motion during export.

## Failure chain and root causes

The bug was not one thing. It was a stack of independent problems:

1. Pen metadata was not available to Composite/Mask.
2. `CompactGraphPlayer` had no Pen renderer.
3. MP4 export was not actually using `CompactGraphPlayer`.
4. DOM snapshot export was capturing before cloned images rendered.
5. CSS mask extraction for export was too brittle.
6. A mask-presence guard incorrectly sampled only the top-left of the mask canvas and treated centered masks as empty.
7. Export mask alpha behavior did not match viewport mask behavior.
8. Full-frame mask application broke transform/pivot semantics for a masked Composite under Motion.

## Prevention notes for future nodes / export work

When adding any new visual node or effect, check all of these paths explicitly:

1. Live editor preview.
2. Composite / Mask interoperability.
3. `resolve*MetaFromEl()` / dataset serialization needs.
4. Static image export.
5. MP4 export DOM snapshot path.
6. Embed/runtime export path (`scripts/export.js`).

Practical lessons:

- If a node needs to be reused as a mask or gradient-like source, serialize stable metadata onto the DOM output instead of trying to reverse-engineer it later from CSS/computed state.
- Do not assume MP4 export and embed/export runtime use the same renderer; verify the actual path first.
- For DOM snapshot export, cloned images must be awaited explicitly.
- For masked/transformed groups, apply the mask as locally as possible to the grouped element rather than as a late full-frame operation.
- If preview treats something as binary visibility, export should mirror that unless a softer alpha behavior is explicitly intended.
- Guard checks on generated canvases should sample the full relevant output, not a corner subset, when content can be centered.
