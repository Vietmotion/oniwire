# DEV LOG - 2026-06-06

## Session Goal
- Stabilize Shape + Gradient render/export behavior and recover reliable still-image export.

## Issues Reported During Session
- Gradient fill on Shape leaked as a full rectangle in render.
- Shape output became invisible in viewport/render after clipping changes.
- Render still output square blocks where star mask was expected.
- Stroke missing in render/export output.
- Export intermittently failed, produced 0 KB files, or froze the app.

## Main Changes Applied

### 1) Shape fill clipping refactor (node runtime)
- Updated Shape fill-input branch to clip via a local frame instead of clipping a full-canvas clone.
- Added local clip-path helpers and safer clone positioning logic.
- Added non-measured-size fallback handling and base-shape fallback when upstream fill geometry is unstable.

Files:
- scripts/nodes/shape.js

### 2) Export renderer geometry parity
- Updated export-side shape drawing to use width/height (not only legacy size).
- Unified rectangle/star/other shape geometry in export canvas renderer to match viewport behavior.

Files:
- scripts/export.js

### 3) Mask bake + mask metadata fixes
- Updated mask shape geometry in main export/capture pipeline to use width/height-aware points.
- Added direct-only mask metadata resolution option to avoid over-applying nested masks to parent containers.
- Updated masked-element bake logic to support direct shape meta while preventing ancestor clipping.
- Added shape stroke metadata (`maskStrokeWidth`, `maskStrokeOpacity`) to shape mask dataset.
- Updated mask rasterization to include stroke for shape masks.

Files:
- oniwire_v0_2_5.html
- scripts/nodes/shape.js

### 4) Still export reliability hardening
- Reworked still-image export path to avoid silent no-op and empty output states.
- Added non-empty blob verification before save/download.
- Reordered and simplified fallback chain to reduce heavy conversion stalls.
- Removed fragile still-image save-handle flow for PNG/JPG in favor of anchor blob download.
- Added in-flight still export guard to prevent overlapping export tasks.
- Added automatic downscale for very large still exports to reduce freeze risk.
- Changed capture order to prefer lighter DOM snapshot path first, then heavier fallbacks only if needed.

Files:
- oniwire_v0_2_5.html

## Result Status at End of Session
- Core render/mask/stroke export logic was significantly stabilized.
- Multiple freeze/0 KB export mitigations were implemented iteratively.
- Remaining user feedback indicated freeze/0 KB behavior still needed final verification on user machine with real project graph and browser download environment.

- This log captures all major edits made in the June 6 stabilization pass across runtime render, export renderer, mask bake pipeline, and still-export reliability handling.
