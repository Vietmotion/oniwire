# DEV LOG - 2026-03-15

## Summary
- Diagnosed and fixed broken PNG/JPG still image export.
- Brought the canvas-based still image renderer up to full parity with v0_1_9 features (Shape fill, Text fill + align, Clone radialMode, dynamic Composite inputPorts).

## What Was Fixed

### 1. Still image export broken (primary fix)
- **Issue:** PNG and JPG exports were not working correctly.
- **Root cause:** `exportCanvasAsImage` branched on `graphUsesBlendModes()`. When no non-normal blend mode was present (the default for most scenes), it fell through to a `html2canvas` DOM-capture path. `html2canvas` cannot reproduce CSS `mask-image` (used by Composite masking), and is unreliable on CDN-load failures. The export produced blank, incorrect, or no output.
- **Fix:** Removed the html2canvas branch entirely. `exportCanvasAsImage` now always delegates to `exportCanvasWithBlend`, which is the proper canvas graph renderer. The canvas renderer already supports all blend modes via `blendMap`, so no functionality is lost.
- **Files:** `oniwire_v0_1_9.html`, `index.html`.

### 2. Shape fill input not rendered in still image export
- **Issue:** Shape connected to a Color or Gradient `fill` input rendered with its flat color param in still export.
- **Fix:** Canvas renderer now checks `getInputCanvas("fill")`. If present, draws fill canvas clipped to shape path using `destination-in` compositing (matching embed runtime behavior).
- **Files:** `oniwire_v0_1_9.html`, `index.html`.

### 3. Text fill input + align not rendered in still image export
- **Issue:** Text connected to `fill` input ignored it; `align` param (left/center/right) had no effect.
- **Fix:** Canvas renderer now:
  - Computes `textBlockWidth` from measured line widths (or `boxWidth` in box mode).
  - Applies `align` to shift `baseX` and compute per-line x-offset (`getLineX`).
  - If `fill` canvas is connected: draws fill canvas, clips to text mask via `destination-in` (matching embed runtime).
  - Unified box-wrap and manual-wrap into a single `getLines()` helper.
- **Files:** `oniwire_v0_1_9.html`, `index.html`.

### 4. Clone radial mode mismatch in still image export
- **Issue:** Clone in radial mode used old formula `(angleStart + angleStep * i)` without 12-o'clock offset, and never used `radialMode` (simple vs advanced) param.
- **Fix:** Canvas renderer now reads `radialMode`:
  - `simple`: evenly spaces N clones in 360°, starting at 12 o'clock (`-90 + evenStep * i`), no `centerX/Y` offset (matches live preview / embed runtime).
  - `advanced`: uses `angleStart - 90 + angleStep * i` with `centerX`/`centerY` offset.
- **Files:** `oniwire_v0_1_9.html`, `index.html`.

### 5. Composite dynamic inputPorts not used in still image export
- **Issue:** Canvas renderer hardcoded `a` and `b` inputs. Composites with extra dynamic ports (added via `+ Input`) silently dropped layers above two.
- **Fix:** Canvas renderer now reads `node.params.inputPorts` (falls back to `["a","b"]`), maps each port to `getInputCanvas`, and draws layers bottom-to-top with blend/opacity applied to all but the bottom-most non-null layer (matching embed runtime logic).
- **Files:** `oniwire_v0_1_9.html`, `index.html`.
