## What Was Fixed
1. Preview ratio to export metadata sync
- Issue: Output ratio could be changed in UI, but exported Lottie still used old dimensions.
- Root cause: `previewBase` object was replaced, while exporter held the old object reference.
- Fix: Mutate `previewBase.w` and `previewBase.h` in place instead of reassigning object.
- Files: `oniwire_v0_1_9.html`, `index.html`.

2. Composite + Motion pivot mismatch in Lottie
- Issue: Composite after text+shape breathed around separate pivots in Lottie.
- Initial fix attempt: shared world anchor on leaf layers (caused frame shift in some cases).
- Final fix: export a parent null controller layer at merged group center and parent child layers under it; apply breath scale only to controller.
- File: `scripts/export.js`.

3. Cache busting for exporter script
- Updated export script query version to ensure browser loads latest exporter changes.
- Files: `oniwire_v0_1_9.html`, `index.html`.

4. Composite node dynamic input ports
- Added per-node `inputPorts` for Composite and dynamic port rendering in editor.
- Added inspector action buttons to append/remove Composite input ports (`+ Input` / `- Input`).
- Updated graph evaluation to support function-based dynamic `inputs`/`outputs` in node defs.
- Updated exporter runtime + Lottie conversion to handle multi-input Composite ordering and blending.
- Files: `scripts/nodes/composite.js`, `oniwire_v0_1_9.html`, `index.html`, `scripts/export.js`.
- Remove action now also cleans wires connected to the removed port and enforces minimum 2 layer inputs.

5. Text node fill input (Color/Gradient)
- Added `fill` input port on Text node.
- Text can now render with:
  - direct Text color parameter (fallback),
  - connected Color node fill,
  - connected Gradient node fill.
- Live preview applies gradient text via CSS text clipping.
- Export pipeline updated so text fill is preserved in:
  - Lottie text raster asset generation,
  - embed runtime canvas rendering.
- Lottie unsupported-node traversal now ignores `Text.fill` validation path to avoid false blocking.
- Files: `scripts/nodes/text.js`, `scripts/export.js`, `oniwire_v0_1_9.html`, `index.html`.

6. Text node align buttons (left/center/right)
- Added new Text param `align` (default `left`) and inspector button-style control for Left/Center/Right.
- Live preview now applies alignment as anchor behavior:
  - left: `x` = text left edge,
  - center: `x` = text center,
  - right: `x` = text right edge.
- Export pipeline updated so rendered text image bounds and embed-runtime canvas drawing respect the same alignment rules.
- Files: `scripts/nodes/text.js`, `scripts/export.js`, `oniwire_v0_1_9.html`, `index.html`.

7. Shape node size range expanded
- Increased Shape `size` slider max from `400` to `1500` so users can scale shapes to fill the screen area in typical ratios.
- Manual number input behavior is unchanged, so users can still type values beyond slider max when needed.
- File: `scripts/nodes/shape.js`.

8. Shape node fill input (Color/Gradient)
- Added `fill` input port on Shape node.
- Preview now uses connected Color/Gradient fill when present, with Shape color param as fallback.
- Export pipeline updated for parity:
  - embed runtime uses fill canvas masked by shape path,
  - Lottie export uses solid vector fill for Color input, and rasterized image asset for Gradient input.
- Lottie dependency traversal now ignores `Shape.fill` upstream branch (same approach as `Text.fill`) to avoid false unsupported-node blocking.
- Files: `scripts/nodes/shape.js`, `scripts/export.js`.

9. Shape selection box for Composite mask usage
- Issue: selecting a Shape node connected only to `Composite.mask` did not show the preview bounding box.
- Root cause: selection box logic relied only on rendered DOM elements tagged with `data-bound-id`; mask-only shape nodes may not render as standalone elements.
- Fix: added fallback bounds calculation for Shape from node params (`x`, `y`, `size`) when no bound DOM target is found.
- Files: `oniwire_v0_1_9.html`, `index.html`.
