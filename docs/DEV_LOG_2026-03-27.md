## What Was Done

### Feature: Multi-Path Shape Manager (Pen Node)

Rewrote the Pen node from a single-path model into a full multi-path system.

**Data model changes (`pen.js` + HTML):**
- `params.paths[]` — array of path objects, each with: `name`, `points[]`, `closed`, `fillColor`, `fillOpacity`, `strokeColor`, `strokeWidth`, `strokeOpacity`, `visible`
- `params.activePath` — index of currently active path
- `params.penSelectedIndex` — selected anchor within active path
- Legacy single-path format auto-migrated via `migratePenNode()`

**Shape Manager inspector panel:**
- Scrollable list of paths with eye toggle, name label, delete button
- Buttons: "+ New Path", "Duplicate", "Clear Points"
- Per-path props panel: Name field, Closed checkbox, Fill/Stroke color + opacity sliders
- Live name update (updates both list row and props title on input)

**Canvas overlay:**
- Ghost paths drawn for all inactive visible paths
- Full handle UI (anchor + in/out tangents) for active path only
- `contextmenu` suppressed on canvas area

### Feature: Illustrator-Style Canvas Interactions

Three distinct mouse behaviors on the pen canvas:
1. **Left-drag on handle** — moves anchor or tangent handle
2. **Left-click near curve** — inserts new point via De Casteljau split (`insertPointOnPenPath`)
3. **Right-click near anchor** — deletes that anchor

**Immediate tangent drag (`newPointHandle` mode):**  
When a new point is appended (or inserted on curve), mouse-hold immediately enters handle-drag so the user can shape the tangent without releasing.

### Feature: Scale Pivot at Bounding Box Center

- `getPathBounds()` in `pen.js` computes bounding box center `{cx, cy}` across all path points + handles
- SVG `<g>` transform: `translate(cx cy) scale(s) translate(-cx -cy)`
- Canvas coordinate math (`penLocalToCanvas`, `canvasToPenLocal`) uses same pivot offset

### Feature: Path Renaming

Name text field in Shape Manager; live-updates the list row label and the props panel title without full re-render.

## Bug Fixes

| Issue | Root Cause | Fix |
|-------|-----------|-----|
| New path stops at 2 points | Bootstrap only appended when `points.length < 2` | Changed to `< 2 \|\| (open && < 3)` |
| No tangent drag on newly created point | Point append did not enter drag state | Added `newPointHandle` kind + `setPointerCapture` after append |
| Stops at 3 on 3rd+ path | `pathOrAdd` mode deferred append to `pointerup`, unreliable | Replaced with bounds-based disambiguation: outside bounding box = append new point, inside = drag whole path |
| Legacy single-path data breaks | Old `points[]` format on root node | `migratePenNode()` auto-upgrades on first access |
| Empty paths restoring default points | `normalizePenPoints` always fell back to defaults | Added `hasExplicitPoints` check — empty array stays empty |

## Key Functions Added/Modified

- `migratePenNode(node)` — upgrades old format to `paths[]`
- `normalizePenPaths(raw)` — normalizes each path object
- `getPenActivePath(node)` — returns active path, ensures `paths[]` exists
- `getPenLocalBounds(node)` — bounding box center in local space
- `penLocalToCanvas` / `canvasToPenLocal` — pivot-aware coordinate transforms
- `drawPenCanvasOverlay()` — ghost + active handle rendering
- `pickPenControl(node, canvasPt)` — anchor/handle hit-test
- `getPenPathCanvasBounds(node, path)` — path bounding box in canvas coords
- `pointInPenPathBounds(canvasPt, bounds, padding)` — padded point-in-box test
- `findPenInsertTarget` / `insertPointOnPenPath` — curve insertion
- `findClosestPenAnchor` — right-click delete target
- `resetPenCanvasState()` — cleanup
