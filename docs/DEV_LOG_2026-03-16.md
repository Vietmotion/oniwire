# DEV LOG - 2026-03-16

## Summary
- Fixed Lottie mode capability gating so Gradient and Ramp are no longer flagged as unsupported in Output mode checks.

## What Was Fixed

### 1. Output node Lottie restriction warning was stale
- **Issue:** Enabling Lottie mode in Output still reported `Gradient` and `Ramp` as unsupported.
- **Root cause:** Lottie allow-list omitted `Gradient` and `Ramp` in editor-side gating.
- **Fix:** Added `Gradient` and `Ramp` to `LOTTIE_ALLOWED_NODE_TYPES`.
- **Files:** `index.html`, `oniwire_v0_1_9.html`.

### 2. Export-side support gate not aligned
- **Issue:** Lottie export type gate could still reject graphs containing gradient/ramp node types.
- **Root cause:** `supportedTypes` set in export traversal omitted `Gradient` and `Ramp`.
- **Fix:** Added `Gradient` and `Ramp` to `supportedTypes` in export gate.
- **Files:** `scripts/export.js`.

## Notes
- This change updates capability checks and messaging parity.
- Existing gradient fill export behavior remains unchanged (current pipeline still resolves gradient fill as implemented in export renderer).

## Pending

## Late Update - Lottie Preview Blank Fix
- **Issue:** Some exported Lottie files loaded with correct metadata/layer counts but rendered blank in external Lottie preview players.
- **Fixes applied in exporter (`scripts/export.js`):
	- Added explicit `w`/`h` on image layers (`ty:2`) for shape/text raster assets to improve player compatibility.
	- Implemented direct `Gradient` layer decoding path for Lottie export (previously allowed by gate, but not decoded into a renderable layer).
	- Added full-canvas gradient raster asset generation for direct Gradient layers.
- **Result:** Lottie payload now includes renderable gradient/image layer dimensions consistently, reducing blank-player outcomes.

## Late Update - Composite Mask Export Fix (Lottie)
- **Issue:** Composite node mask input still did not clip output in exported Lottie (mask appeared as regular layer behavior instead of matte behavior).
- **Root cause:** Initial implementation used blend/parent behavior (`bm`) instead of Lottie track matte semantics.
- **Fixes applied in exporter (`scripts/export.js`):
	- Replaced mask handling with Lottie track matte flow (`tt` on masked layer, matte layer directly above).
	- Wrapped composite content into a precomp asset so the mask clips the full composite result, not just one child layer.
	- Restored original composite motion grouping behavior for non-mask paths while preserving blend/opacity logic.
- **Result:** Composite mask now exports as a true matte pipeline compatible with Lottie player behavior.
