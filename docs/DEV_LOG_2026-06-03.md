# DEV LOG - 2026-06-03

## Summary

Today focused on stabilization and release hardening after a critical render/export regression, plus quality-of-life graph UX upgrades and typography pipeline improvements.

The most critical outcomes:
- Export rendering parity was restored between preview, PNG/JPG, and MP4.
- Mask + filter + image edge cases were stabilized.
- Wire editing UX gained auto-insert and reliable right-click delete.
- A new Font node was introduced and connected to Text for custom uploaded fonts.

## Critical Fixes

### 1. Export pipeline regression (critical)

Problem:
- Render output became broken after filter-bake changes, with PNG/MP4 diverging from preview in complex scenes.

Root causes:
- html2canvas ignores CSS filter in several cases.
- Capture order drift between still export and MP4 frame capture.
- Over-aggressive deterministic masked-element baking in mixed-content nodes.

Fixes:
- Enforced canonical capture order in capture helpers:
  clone -> wait images -> bake object-fit images -> bake filtered elements -> bake masked elements -> html2canvas.
- Added robust filter bake fallback for `filter: url(#...)` SVG component-transfer paths via pixel remap.
- Added deterministic mask-bake guard (`hasVisibleNonImageContent`) to avoid splitting mixed painted content.
- Kept mask baking enabled consistently in still and MP4 paths (`disableBakeMaskedElements: false`).

Impact:
- PNG/JPG and MP4 now match preview much more reliably in layered mask/filter scenes.

### 2. Wire interaction reliability

Problem:
- Right-click wire delete intermittently failed due to SVG wire layer being non-interactive (`pointer-events: none`).

Fixes:
- Moved wire hit handling to editor-level geometry hit-testing.
- Added nearest-wire detection over bezier path sampling.
- Added right-click wire delete and right-click input unstring handling at editor/input level.

Impact:
- Wire deletion and unstringing are now reliable regardless of SVG pointer-event settings.

## Feature Work

### 1. Auto-connect insert-on-wire (Blender-style)

- Added drag-over wire candidate detection.
- Added glow highlight for drop candidate wire.
- On node drop over wire, auto-splits old connection and inserts node between endpoints.

### 2. Text default centering

- New Text nodes now spawn centered in preview with centered alignment.

### 3. Font node + Text font input

- Added new `Font` generator node.
- Font node supports custom font upload (`ttf`, `otf`, `woff`, `woff2`, `ttc`) and runtime `FontFace` loading.
- Text node now exposes optional `font` input in addition to `fill`.
- When Font node is connected:
  - Text font family is driven by Font node.
  - Text weight is driven by Font node "Font family" selector.
  - Text node's local Font and Font family controls are disabled and visibly greyed out.
