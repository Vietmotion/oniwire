## Critical errors and freeze causes (important)
- Freeze cause #1: Corrupted script structure in `oniwire_v0_2_2.html`.
  - Symptom: App appeared frozen; diagnostics showed parse errors near file end (`})();`) and mid-file (`')' expected`).
  - Root cause: A patch operation inserted valid code into the wrong function block, breaking brace structure.
  - Impact: JS parse failure halted runtime initialization, so editor became non-responsive.

- Freeze cause #2: Stray code fragment inserted into unrelated helper functions.
  - Symptom: Additional compile errors around `renderLayerToDataUrl` and `dataUrlToCanvas`.
  - Root cause: Mask-meta snippet was accidentally injected into rendering helpers (`measureLayerBounds`, `renderLayerToDataUrl`, `dataUrlToCanvas`).
  - Impact: Runtime could not evaluate script; app load/update path broke.

- Critical logic bug (non-freeze): Shape mask metadata mismatch.
  - Symptom: Connecting Shape into Composite mask could blank output.
  - Root cause: Shape wrote `maskWidth` / `maskHeight` while mask resolver returned `size`, but shape mask builder consumed `width` / `height`.
  - Impact: Invalid geometry values (`NaN`) could generate unusable mask output.

## Fixes applied for the critical issues
- Restored corrupted blocks to valid structure in:
  - `measureLayerBounds`
  - `renderLayerToDataUrl`
  - `dataUrlToCanvas`
- Re-applied mask meta fix in the correct place only (`resolveMaskMetaFromEl`).
- Re-ran diagnostics until no compile errors remained.

## Prevention checklist (for future patches)
- Always run file diagnostics immediately after any large patch to `oniwire_v0_2_2.html`.
- If one patch touches multiple distant regions, verify each touched function block boundaries (`{}`) before next patch.
- Prefer single-purpose patches with unique context anchors (function-level) when editing large HTML+script files.
- After patching, re-open and inspect full modified functions (not just changed lines) to catch misplaced insertions.
- When data contracts change (`size` vs `width/height`), verify producer + consumer fields together.
